The Problem

Why Standard FOC Falls Short for Robotics

Field-Oriented Control delivers smooth sinusoidal commutation, but real-world quasi-direct-drive actuators suffer from systematic disturbances that standard PI controllers cannot eliminate:

  • Cogging torque — Permanent magnet interaction with stator teeth creates position-dependent torque ripple (up to 5% of rated torque)
  • Thermal drift — Winding resistance increases ~0.4%/K, shifting PI operating points as the motor heats
  • Gearbox friction — Cycloidal reducers exhibit speed-dependent friction and backlash that varies with load direction
  • Manufacturing variation — Phase resistance and inductance mismatch between motors of the same SKU

These effects compound to a 32% cost-of-transport penalty compared to a perfectly compensated drive. The Adaptive Actuator Intelligence (AAI) system eliminates these losses through on-device learning.

AAI Architecture

ISR Timing Budget

The learned compensator executes inside the 20 kHz TIM1 update ISR. Every microsecond matters. The total budget is 50 us per cycle, with the FOC pipeline consuming approximately 28 us:

The AAI compensator sits between the Park transform and PI controllers, injecting correction voltages deltaVd and deltaVq into the dq-frame. This placement allows it to compensate for all disturbances simultaneously without disturbing the stability margins of the PI loops.

C1 — Self-Profiling

Automatic Motor Characterization

On first boot (or on command), the TITAN_FOC runs a comprehensive self-profiling sequence that measures the connected motor's electrical and mechanical parameters:

  • Phase resistance (R) — DC injection at multiple current levels, temperature-compensated via NTC
  • Phase inductance (L_d, L_q) — High-frequency injection at locked rotor, measured via current response slope
  • Flux linkage (lambda_pm) — Back-EMF measurement during constant-speed open-loop spin
  • Cogging torque map — 4096-bin position-indexed torque measurement (one full electrical revolution at 14-bit encoder resolution)
  • Encoder offset — Automatic alignment of electrical angle zero to encoder index

All parameters are stored in flash and used to initialize the PI gains, feedforward terms, and the AAI compensator's input normalization.

typedef struct { float R_phase; // Ohms, per-phase float L_d, L_q; // Henries, dq-frame float lambda_pm; // Wb, PM flux linkage int16_t cogging[4096]; // Q15 torque map uint16_t enc_offset; // Encoder electrical zero float temp_ref; // Reference temperature (C) } motor_profile_t;

C2 — Learned Compensator

Fixed-Point Neural Network

The core of AAI is a tiny neural network running entirely in fixed-point Q15.16 arithmetic on the Cortex-M4's integer pipeline. This avoids FPU contention with the main FOC float math:

  • Inputs (5) — theta_e (electrical angle), omega_e (electrical velocity), I_d, I_q, temperature
  • Hidden layer — 16 neurons, ReLU activation, Q15 weights
  • Outputs (2) — deltaV_d, deltaV_q correction voltages
  • Execution — Uses CORDIC for sin/cos lookup, all MAC operations in Q15, total latency under 5 us
// Q15 MAC: 16 neurons x 5 inputs = 80 multiply-accumulates for (int i = 0; i < 16; i++) { int32_t acc = bias[i]; for (int j = 0; j < 5; j++) acc += (int32_t)w[i][j] * input[j]; // Q15 * Q15 = Q30 hidden[i] = (acc > 0) ? (acc >> 15) : 0; // ReLU + Q30->Q15 }

Weights are trained offline via the Digital Twin (C3) and uploaded over CAN-FD. On-device fine-tuning uses a simplified online gradient descent with a learning rate of 2^-12 to track slow parameter drift (thermal, wear).
